# Component Contracts

本页规则以组件驱动设计和可维护 API 约束为基线。

## 契约内容

- 组件解决什么问题
- 暴露哪些输入、状态、槽位和事件
- 哪些可访问性职责由组件内建承担
- 哪些行为属于业务层而不应下沉

## 设计原则

- API 围绕语义和意图，而不是围绕临时视觉微调
- 变体数量应可解释，避免为单个页面扩充永久参数
- 默认、悬停、聚焦、禁用、错误、加载等状态应有一致定义
- 组件不应把业务文案、接口数据结构或路由逻辑硬编码进去

## 常见修正方向

- 把一堆布尔参数收敛成清晰的状态或变体
- 把难以组合的大组件拆成基础组件和组合层
- 把可访问性行为内建，而不是要求调用方手工补齐
