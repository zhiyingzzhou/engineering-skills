# DOM Testing

本页规则以 Testing Library 的 Guiding Principles 为基线。

## 查询优先级

- 优先按 role 和 accessible name 查询
- 其次按 label、文本或占位提示查询
- 最后才考虑稳定的 `data-testid`

## 编写原则

- 用接近真实用户的交互触发行为
- 断言用户能看到、读到或操作到的结果
- 不把内部 state、私有函数或 DOM 结构细节当主断言
- 大型快照只能作辅助手段，不能替代行为断言

## 常见修正方向

- 把脆弱的类名选择器改成语义查询
- 把“调用了某函数”改成“界面发生了什么变化”
- 把多场景巨型测试拆成几个小而清晰的案例
