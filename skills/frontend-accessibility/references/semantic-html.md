# Semantic HTML

本页规则以 MDN 的无障碍 HTML 指南为基线。

## 核心原则

- 先用原生元素表达职责，再考虑 ARIA
- 操作用 `button`，导航用 `a`，表单输入用原生表单控件
- 页面应有清晰的标题层级和主要 landmark
- 数据表格只在确实是表格数据时使用

## 语义检查

- 标题层级是否连续且反映内容结构
- `main`、`nav`、`header`、`footer`、`aside` 是否用于真实区域
- 图片、图标和装饰是否有正确的文本替代策略
- 列表、定义列表、表格是否表达真实关系，而不是只为样式服务

## 常见修正方向

- 把可点击 `div` 改回 `button` 或 `a`
- 把只用于布局的标签从语义结构中降噪
- 为控件补齐可见标签或程序化名称
- 为复杂内容补充真实标题，而不是只靠粗体文本
